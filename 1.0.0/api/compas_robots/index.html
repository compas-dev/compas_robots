    compas_robots ¤

    This package provides the basic infrastructure for working with robots in COMPAS.

    The root of a robot model is the RobotModel class, which describes a robot consisting of a set of link elements, and a set of joint elements connecting the links together.

    Configuration ¤

    Configuration(
    joint_values: list[float] | None = None,
    joint_types: list[JointType] | None = None,
    joint_names: list[str] | None = None,
    **kwargs
)

    Represents the configuration of a robot based on the state of its joints. If the names of joints are also provided, the configuration behaves as a dictionary of joint name-value pairs.

    This concept is also refered to as Joint State.

    Parameters:

    • joint_values (list[float] | None, default: None ) –

      Joint values expressed in radians or meters, depending on the respective type.

    • joint_types (list[JointType] | None, default: None ) –

      Joint types, e.g. a list of JointType.REVOLUTE for revolute joints.

    • joint_names (list[str] | None, default: None ) –

      List of joint names.

    Attributes:

    • joint_values (list[float]) –

      Joint values expressed in radians or meters, depending on the respective type.

    • joint_types (list[JointType]) –

      Joint types, e.g. a list of JointType.REVOLUTE for revolute joints.

    • joint_names (list[str]) –

      List of joint names.

    • data (dict) –

      The data representing the configuration.

    • prismatic_values (list[float]) –

      Prismatic joint values in meters.

    • revolute_values (list[float]) –

      Revolute joint values in radians.

    Examples:

    >>> config = Configuration.from_revolute_values([math.pi / 2, 0.0, 0.0])
>>> config.joint_values
[1.5707963267948966, 0.0, 0.0]

    >>> config = Configuration.from_prismatic_and_revolute_values([8.312], [math.pi / 2, 0.0, 0.0, 0.0, 2 * math.pi, 0.8])
>>> str(config)
'Configuration((8.312, 1.571, 0.000, 0.000, 0.000, 6.283, 0.800), (2, 0, 0, 0, 0, 0, 0))'

    >>> from compas_robots.model import Joint
>>> config = Configuration([math.pi / 2, 3.0, 0.1], [Joint.REVOLUTE, Joint.PRISMATIC, Joint.PLANAR])
>>> str(config)
'Configuration((1.571, 3.000, 0.100), (0, 2, 5))'

    iter_differences ¤
    iter_differences(other: Configuration) -> Iterator[float]

    Generator over the differences to another Configuration's joint_values.

    If the joint type is revolute or continuous, the smaller difference (+/- 2*\pi) is calculated.

    Parameters:

    • other (Configuration) –

      The configuration to compare to.

    Yields:

    • The next difference to the `Configuration`'s joint_values. –

    Raises:

    • ValueError –

      If the configurations are not comparable.

    Examples:

    >>> from compas.geometry import allclose
>>> c1 = Configuration.from_revolute_values([1, 0, 3])
>>> c2 = Configuration.from_revolute_values([1, 2 * pi, 4])
>>> allclose(c1.iter_differences(c2), [0.0, 0.0, -1.0])
True
>>> c1 = Configuration.from_revolute_values([1, 0, 3])
>>> c2 = Configuration.from_revolute_values([1, -2 * pi - 0.2, 4])
>>> allclose(c1.iter_differences(c2), [0.0, 0.2, -1.0])
True

    RobotModel ¤

    RobotModel(name=None, joints=(), links=(), materials=(), **kwargs)

    RobotModel is the root element of the model.

    Instances of this class represent an entire robot as defined in an URDF structure.

    In line with URDF limitations, only tree structures can be represented by this model, ruling out all parallel robots.

    Attributes:

      List of links of the robot.

    • materials (list[Material]) –

    axis: Vector | None = None,
    limit: tuple[float, float] | None = None,
    **kwargs
) -> Joint

    Adds a joint to the robot model.

    Provides an easy way to programmatically add a joint to the robot model.

    Parameters:

    • name (str) –

      The name of the joint

    • type (int) –

      The joint type, e.g. Joint.REVOLUTE

    • parent_link (Link) –

      The joint's parent link.

    • child_link (Link) –

      The joint's child link.

    • origin (Frame | None, default: None ) –

      The joint's origin frame.

    • axis (Vector | None, default: None ) –

      The joint's axis.

    • limit (tuple[float, float] | None, default: None ) –

      The lower and upper limits of the joint (used for joint types Joint.REVOLUTE or Joint.PRISMATIC)

    • **kwargs –

      The keyword arguments (kwargs) collected in a dict. These allow using non-standard attributes absent in the URDF specification.

    Returns:

    • Joint –

      The newly created joint instance.

    Raises:

    • ValueError –

      If the joint name is already used in the chain.

    Examples:

    >>> from compas.geometry import Frame
>>> robot = RobotModel("robot")
>>> parent_link = robot.add_link("link0")
>>> child_link = robot.add_link("link1")
>>> origin = Frame.worldXY()
>>> axis = (1, 0, 0)
>>> j = robot.add_joint("joint1", Joint.CONTINUOUS, parent_link, child_link, origin, axis)
